How This Works

Adapted from Karpathy's autoresearch pattern — autonomous loop, different domain.

Cycle A

New Models

Firecrawl searches for model announcements, paper releases, and HuggingFace uploads from the last 24 hours. Cross-references against the registry.

Cycle B

New Companies

Searches for AI startup funding rounds, new lab launches, and company formations. Filters out fine-tuners and API wrappers using strict classification rules.

Cycle C

Status Changes

Monitors acquisitions, shutdowns, mergers, and pivots. Catches when a company stops training original models or gets absorbed.

Cycle D

Leaderboard Validation

Scrapes HuggingFace leaderboards and benchmark results. Identifies new model families from unknown organisations.

Cycle E

Regional Deep Dives

Weekly rotation through China, Europe, Asia-Pacific, and Middle East. Catches companies that don't show up in English-language searches.

Qualification

Strict Criteria

Must pre-train on own architecture with own compute. Must produce model weights. No fine-tuners, no API wrappers, no inference-only providers.
