Methodology

Five autonomous cycles.
One complete picture.

Adapted from Karpathy's autoresearch pattern. Instead of optimising a neural network, this loop optimises registry completeness.

Cycle A

New Models

Firecrawl searches for model announcements, paper releases, and HuggingFace uploads from the last 24 hours.

Cycle B

New Companies

AI startup funding rounds, new lab launches, company formations. Filters out fine-tuners and API wrappers.

Cycle C

Status Changes

Acquisitions, shutdowns, mergers, pivots. Catches when a company stops training or gets absorbed.

Cycle D

Leaderboard Check

Scrapes HuggingFace leaderboards and benchmark results. Identifies unknown model families.

Cycle E

Regional Deep Dives

Weekly rotation: China, Europe, Asia-Pacific, Middle East. Catches companies invisible to English-language search.

Criteria

Strict Qualification

Must pre-train on own architecture with own compute. Must produce model weights. No exceptions.

About This Project

Built by Project Zero as an orchestration layer for understanding the AI landscape. Powered by Firecrawl for web intelligence and Claude for autonomous research.

The registry is maintained through autonomous search cycles, cross-referenced against multiple sources, and validated before any change is committed.
